# 캐디하루 MVP 제품 요구사항

- 문서 상태: 초안 확정
- 작성일: 2026-06-18
- 대상: 웹 MVP

## 1. 제품 개요

캐디하루는 캐디 개인이 하루 동안 얻은 캐디피와 오버피를 라운드 단위로 기록하고, 월간 및 연간 수입을 캘린더와 통계로 확인하는 개인 수입 장부다.

관리자, 골프장, 다른 캐디와 데이터를 공유하지 않는다. 각 사용자는 자신의 기록만 생성, 조회, 수정, 삭제할 수 있다.

## 2. 제품 목표

- 근무 직후 10초 안에 한 라운드의 수입을 기록한다.
- 하루 여러 라운드의 수입을 날짜별로 합산한다.
- 월간 캘린더에서 일별 수입을 빠르게 파악한다.
- 월간 및 연간 수입 추이를 확인한다.
- 급여 데이터를 로컬 영구 저장소가 아닌 서버에 안전하게 보관한다.
- 향후 React Native 앱에서도 같은 백엔드와 도메인 모델을 사용할 수 있게 한다.

## 3. 대상 사용자와 전제

- 국내에서 한 골프장에 소속되어 근무하는 개인 캐디
- 초기 예상 사용자: 100명 이상
- 하루 두 번 이상 라운드를 도는 경우가 흔하다.
- 캐디피는 소속 골프장에서 대체로 고정되어 있다.
- 수입은 현장에서 현금 또는 계좌이체로 바로 지급된다.
- 미정산 관리는 필요하지 않다.
- 모바일 웹 사용을 우선한다.

## 4. MVP 범위

### 4.1 인증과 온보딩

- 카카오 로그인만 제공한다.
- 최초 로그인 후 다음 정보를 설정한다.
  - 소속 골프장명
  - 기본 캐디피
  - 오버피 빠른 선택값
- 오버피 빠른 선택 기본값은 `없음`, `1만원`, `2만원`, `3만원`, `직접 입력`이다.
- 카카오 계정에서는 서비스 식별에 필요한 최소 정보만 사용한다.

### 4.2 수입 기록

- 오늘과 과거 날짜에만 라운드를 추가할 수 있다.
- 하루에 라운드를 여러 건 추가할 수 있다.
- 라운드별 입력 항목은 다음과 같다.
  - 근무 날짜
  - 캐디피
  - 오버피
  - 결제 수단: 현금 또는 계좌이체
  - 선택 메모
- 캐디피에는 프로필의 기본 캐디피를 자동 입력한다.
- 오버피 기본값은 0원이다.
- 한 라운드 수입은 `캐디피 + 오버피`로 계산한다.
- 하루 수입은 같은 날짜의 모든 라운드 수입 합계다.
- 월 수입과 연 수입은 해당 기간의 라운드 수입 합계다.
- 기존 라운드를 수정하거나 삭제할 수 있다.
- 결제 수단은 기록 상세에서만 보여주며 통계 집계 항목으로 사용하지 않는다.
- 설정에서 소속 골프장을 변경해도 과거 기록의 골프장명은 바뀌지 않는다.

### 4.3 월간 캘린더

- 캘린더는 일요일부터 시작한다.
- 상단에 현재 연월, 월 총수입, 월 총 라운드 수를 표시한다.
- 이전 달과 다음 달로 이동할 수 있다.
- 날짜 셀에는 `15만`, `32만` 형식의 축약 일 수입을 표시한다.
- 기록이 없는 날짜에는 금액을 표시하지 않는다.
- 오늘은 외곽선으로, 선택한 날짜는 브랜드 색상으로 구분한다.
- 날짜를 선택하면 캘린더 아래에 라운드별 상세와 하루 합계를 표시한다.
- 선택한 날짜에 라운드를 추가하는 하단 고정 버튼을 제공한다.

### 4.4 통계

통계 화면은 `월간`과 `연간` 탭으로 구분한다.

월간 통계:

- 월 총수입
- 월 총 라운드 수
- 캐디피 합계
- 오버피 합계
- 일별 수입 막대그래프

연간 통계:

- 연 총수입
- 연 총 라운드 수
- 월별 수입 추이
- 월별 캐디피와 오버피 구성
- 연도 이동

현금 수입과 계좌이체 수입은 별도 통계로 제공하지 않는다.

### 4.5 설정과 데이터 소유권

- 소속 골프장명 변경
- 기본 캐디피 변경
- 오버피 빠른 선택값 변경
- 화면 모드 설정: 시스템 설정, 라이트, 다크
- 기간을 선택한 CSV 내보내기
- 계정과 전체 데이터 삭제
- 기본값 변경은 기존 수입 기록에 영향을 주지 않는다.

CSV에는 다음 열을 포함한다.

- 날짜
- 해당 날짜의 라운드 순서
- 골프장명
- 캐디피
- 오버피
- 결제 수단
- 라운드 합계
- 메모

### 4.6 PWA

PWA는 MVP의 가장 낮은 우선순위로 구현한다.

- 홈 화면 설치
- 캐디하루 앱 아이콘 및 manifest
- 앱 기본 화면과 정적 리소스 캐싱
- 급여 데이터의 로컬 영구 저장 금지
- 오프라인 상태와 저장 불가 상태 표시
- 푸시 알림과 수입 기록 알림은 제공하지 않는다.

## 5. 정보 구조

하단 내비게이션은 세 개의 탭으로 구성한다.

1. 캘린더
2. 통계
3. 설정

로그인하지 않은 사용자는 카카오 로그인 화면으로 이동한다. 최초 로그인 후 온보딩을 완료해야 캘린더에 진입할 수 있다.

## 6. 핵심 사용자 흐름

### 6.1 첫 사용

1. 카카오로 로그인한다.
2. 소속 골프장명을 입력한다.
3. 기본 캐디피를 입력한다.
4. 오버피 빠른 선택값을 확인하거나 수정한다.
5. 이번 달 캘린더로 이동한다.

### 6.2 라운드 추가

1. 캘린더에서 오늘 또는 과거 날짜를 선택한다.
2. `라운드 추가` 버튼을 누른다.
3. 기본 캐디피가 입력된 바텀시트가 열린다.
4. 오버피를 선택하거나 직접 입력한다.
5. 현금 또는 계좌이체를 선택한다.
6. 필요하면 메모를 입력한다.
7. 라운드 합계를 확인하고 저장한다.
8. 서버 저장이 완료되면 날짜와 월 합계가 갱신된다.

### 6.3 저장 실패

1. 저장 중 중복 요청을 방지한다.
2. 서버 응답이 실패하면 성공 상태로 표시하지 않는다.
3. 입력값은 현재 화면 메모리에 유지한다.
4. 오류 원인과 재시도 버튼을 표시한다.
5. 영구 로컬 캐시에는 수입 기록이나 입력 초안을 저장하지 않는다.

- 카카오 로그인 버튼
- 개인정보 처리방침 및 이용약관 링크

### 7.2 캘린더

- 월 이동 헤더
- 월 총수입과 총 라운드 수
- 7열 월간 캘린더

- 선택 날짜의 하루 합계
- 라운드 추가 버튼
- 라운드별 수정 및 삭제 명령

### 7.3 라운드 입력 바텀시트

- 날짜와 자동 계산된 라운드 순서
- 원 단위 숫자 키패드를 여는 캐디피 입력
- 오버피 빠른 선택과 직접 입력
- 현금과 계좌이체 세그먼트 컨트롤
- 선택 메모
- 실시간 라운드 합계
- 저장 버튼과 저장 중 상태

두 번째 이후 라운드에도 기본 캐디피를 사용한다. 라운드 순서는 같은 날짜의 기록을 생성 시각 순으로 정렬해 화면에서 계산한다.

### 7.4 통계

- 기간 이동

- 기간별 막대그래프
- 그래프 항목 선택 시 정확한 금액 표시

### 7.5 설정

- 현재 카카오 계정
- 소속 골프장
- 기본 캐디피

## 8. 디자인 시스템

### 8.1 방향

- 밝고 친근하지만 업무 기록에 적합한 절제된 UI
- 흰색과 에메랄드 계열 초록색 중심
- 네이버 대표 초록색과 유사한 인상 회피
- 야외 밝은 환경에서도 읽을 수 있는 높은 명암 대비
- 모바일 기준 최소 44px 터치 영역
- 장식보다 금액 가독성과 입력 속도 우선
- 금액은 크게, 부가 정보는 작게 표시
- 컴포넌트에서 원시 색상값 대신 시맨틱 토큰 사용

보조 색상은 결제 수단과 시스템 상태 구분에만 제한적으로 사용한다.

- 현금: 앰버
- 계좌이체: 블루
- 오류: 레드
- 오버피: 브랜드 그린 강조
- 비활성 및 기록 없음: 중립 회색

### 8.3 브랜드 아이콘

- 성별을 특정하지 않는 사람 상반신 실루엣
- 넓은 챙의 캐디 모자
- 얼굴 내부 세부 묘사 제외
- 흰색 심볼과 에메랄드 그린 배경
- 작은 앱 아이콘 크기에서 모자와 사람이 구분되어야 한다.
- 앱 아이콘에는 글자를 넣지 않는다.
- 워드마크 `캐디하루`는 심볼과 분리한다.

## 9. 데이터 모델

### 9.1 profiles

| 필드 | 형식 | 설명 |
| --- | --- | --- |
| user_id | uuid | `auth.users.id`, 기본 키 |
| course_name | text | 현재 소속 골프장 |
| default_caddie_fee | integer | 기본 캐디피, 원 단위 |
| over_fee_presets | integer[] | 오버피 빠른 선택값 |
| theme_mode | enum | system, light, dark |
| created_at | timestamptz | 생성 시각 |
| updated_at | timestamptz | 수정 시각 |

### 9.2 round_entries

| 필드 | 형식 | 설명 |
| --- | --- | --- |
| id | uuid | 기본 키 |
| user_id | uuid | 기록 소유 사용자 |
| work_date | date | 근무 날짜 |
| caddie_fee | integer | 캐디피, 원 단위 |
| over_fee | integer | 오버피, 원 단위 |
| payment_method | enum | cash, transfer |
| course_name_snapshot | text | 기록 당시 골프장명 |
| memo | text, nullable | 선택 메모 |
| created_at | timestamptz | 생성 시각 |
| updated_at | timestamptz | 수정 시각 |

금액은 부동소수점이 아닌 원 단위 정수로 저장한다. `caddie_fee`와 `over_fee`에는 0 이상의 값만 허용한다. 날짜 조회를 위해 `(user_id, work_date)` 복합 인덱스를 둔다.

별도 `work_days` 테이블은 만들지 않는다. 일별, 월별, 연간 값은 `round_entries`를 날짜 기준으로 집계한다.

## 10. 보안과 개인정보

- 모든 공개 데이터 테이블에 Supabase RLS를 활성화한다.
- 사용자는 `auth.uid() = user_id`인 행만 조회, 생성, 수정, 삭제할 수 있다.
- 사용자가 변경할 수 있는 인증 메타데이터를 권한 판단에 사용하지 않는다.
- Supabase service role 키는 브라우저 번들에 포함하지 않는다.
- 계정 삭제는 인증된 서버 측 기능에서 처리한다.
- 개인정보 처리방침에 수집 정보, 사용 목적, 보관 기간, 삭제 절차를 명시한다.
- 로그에 수입 금액, 메모, 인증 토큰을 출력하지 않는다.
- 클라이언트의 영구 저장소에 수입 데이터를 보관하지 않는다.

## 11. 기술 구조

- 프레임워크: Next.js App Router, React, TypeScript
- 스타일: vanilla-extract와 시맨틱 디자인 토큰
- 서버 상태: TanStack Query
- 패키지 관리자: pnpm
- 인증 및 데이터베이스: Supabase Auth, PostgreSQL, RLS
- 웹 호스팅: Cloudflare Workers 무료 플랜 우선
- 소스 관리: GitHub
- CI: GitHub Actions
- 컴파일: Next.js 기본 SWC 사용
- Babel: 필수 플러그인이 생기기 전에는 사용하지 않음
- 번들러: Next.js 기본 구성을 우선하고 불필요한 Webpack 사용자 설정을 추가하지 않음

웹 MVP는 단일 애플리케이션으로 시작한다. React Native 개발을 시작할 때 도메인 타입, API 계약, 디자인 원시 토큰을 별도 패키지로 추출한다. 웹용 vanilla-extract 컴포넌트와 React Native UI 컴포넌트는 직접 공유하지 않는다.

## 12. 배포와 무료 플랜 제약

- Supabase Free는 초기 100명 규모에 충분한 용량을 제공하지만 자동 백업이 없고 비활성 프로젝트가 일시 중지될 수 있다.
- 실제 운영에서 데이터 복구 보장이 필요해지는 시점에는 Supabase 유료 플랜을 우선 검토한다.
- Vercel Hobby는 비상업적 개인 프로젝트에 제한되므로 공개 상용 운영의 기본 호스팅으로 사용하지 않는다.
- Cloudflare Workers의 Next.js 배포는 OpenNext 어댑터 호환성을 배포 전 검증한다.
- 비용보다 데이터 복구와 운영 안정성이 중요해지는 시점을 별도 출시 기준으로 둔다.

## 13. GitHub Actions

Pull Request와 기본 브랜치 푸시에서 다음 작업을 수행한다.

1. 의존성 고정 설치
2. lint
3. TypeScript typecheck
4. 단위 테스트
5. 프로덕션 빌드

Supabase 마이그레이션과 RLS 정책은 코드로 버전 관리한다. 데이터베이스 변경은 마이그레이션 검증 없이 운영 환경에 직접 적용하지 않는다.

## 16. 완료 기준

- 사용자는 카카오 로그인과 최초 설정을 완료할 수 있다.
- 사용자는 오늘과 과거 날짜에 여러 라운드를 기록할 수 있다.
- 캐디피 기본값과 오버피 빠른 선택이 올바르게 적용된다.
- 캘린더의 일 합계와 월 합계가 라운드 데이터와 일치한다.
- 월간 및 연간 통계가 같은 원본 데이터에서 정확히 계산된다.
- 사용자는 자신의 데이터만 접근할 수 있다.
- 라이트, 다크, 시스템 모드가 정상 동작한다.
- CSV 내보내기 결과가 화면의 기록과 일치한다.
- 계정 삭제 후 인증 계정과 모든 사용자 데이터가 제거된다.
- 저장 실패 시 입력값이 즉시 사라지지 않고 재시도할 수 있다.
- 모바일 화면에서 주요 조작 영역이 44px 이상이며 텍스트와 컨트롤이 겹치지 않는다.
